# Contributing to UcuzToken Documentation

Thank you for your interest in contributing to UcuzToken documentation!

## 🤝 How to Contribute

### Reporting Issues
If you find any errors or have suggestions:
1. Check existing issues first
2. Create a new issue with detailed description
3. Include examples if applicable

### Documentation Improvements
We welcome improvements to:
- Code examples
- Translations
- API documentation
- Usage guides
- Error handling examples

### Submitting Changes

1. Fork the repository
2. Create a new branch (`git checkout -b feature/improvement`)
3. Make your changes
4. Test all code examples
5. Commit your changes (`git commit -m 'Add: description'`)
6. Push to the branch (`git push origin feature/improvement`)
7. Open a Pull Request

## 📝 Guidelines

### Code Examples
- Ensure all code examples work
- Include error handling
- Add comments for complex logic
- Follow language-specific best practices

### Documentation Style
- Use clear, concise language
- Include both Turkish and English versions
- Add code examples where applicable
- Keep formatting consistent

### Commit Messages
Use conventional commits:
- `Add: new feature or example`
- `Fix: bug fix or correction`
- `Update: improvement to existing content`
- `Docs: documentation only changes`

## 🌍 Translations

We welcome translations to other languages. Please:
1. Create a new file with language code (e.g., `README.de.md`)
2. Translate all content accurately
3. Maintain the same structure
4. Submit a PR

## 📞 Questions?

Contact us via:
- WhatsApp support
- GitHub Issues
- [ucuztoken.com](https://ucuztoken.com)

Thank you for helping improve UcuzToken documentation!
